                                                # Pinia

                                                Vuex和Pinia都能用于网站应用的状态管理，
                                                目前官方更推荐使用 Pinia。

                                                Pinia官方网站open in new window
                                                Pinia 是 Vue 的专属状态管理库，它允许你跨组件或页面共享状态。如果你熟悉组合式 API 的话，你可能会认为可以通过一行简单的 export const state = reactive({}) 来共享一个全局状态。

                                                # 开始

                                                如果你的应用使用的 Vue 版本低于 2.7，你还需要安装组合式 API 包：@vue/composition-api

                                                # 安装

                                                yarn add pinia
# 或者使用 npm
npm install pinia

                                                # 基础使用

                                                创建一个 pinia 实例 (根 store) 并将其传递给应用

                                                1. 获取 createPinia
                                                2. 创建 pinia 实例 
                                                import { createApp } from 'vue'
import { createPinia } from 'pinia'
import App from './App.vue'

const pinia = createPinia()
const app = createApp(App)

app.use(pinia)
app.mount('#app')

                                                如果你使用的是 Vue 2，你还需要安装一个插件PiniaVuePlugin，并在应用的根部注入创建的 pinia

                                                import { createPinia, PiniaVuePlugin } from 'pinia'

Vue.use(PiniaVuePlugin)
const pinia = createPinia()

new Vue({
  el: '#app',
  // 其他配置...
  // ...
  // 请注意，同一个`pinia'实例
  // 可以在同一个页面的多个 Vue 应用中使用。 
  pinia,
})

                                                # 基础案例

                                                下面就是 pinia API 的基本用法

                                                // stores/counter.js
import { defineStore } from 'pinia'

export const useCounterStore = defineStore('counter', {
  state: () => {
    return { count: 0 }
  },
  // 也可以这样定义
  // state: () => ({ count: 0 })
  actions: {
    increment() {
      this.count++
    },
  },
})

                                                然后你就可以在一个组件中使用该 store 了

                                                <script setup>
import { useCounterStore } from '@/stores/counter'
const counter = useCounterStore()
counter.count++
// 自动补全！ ✨
counter.$patch({ count: counter.count + 1 })
// 或使用 action 代替
counter.increment()
</script>
<template>
  <!-- 直接从 store 中访问 state -->
  <div>Current Count: {{ counter.count }}</div>
</template>

                                                # STORE

                                                Store是通过defineStore()定义的。
                                                他的第一个参数是一个仓库的名字。
                                                defineStore() 的第二个参数可接受两类值：Setup 函数或 Option 对象。

                                                import { defineStore } from 'pinia'

export const useStore = defineStore('myStore', {
  // 其他配置...
})

                                                # Option Store

                                                与 Vue 的选项式 API 类似，我们也可以传入一个带有 state、actions 与 getters 属性的 Option 对象

                                                export const useCounterStore = defineStore('counter', {
  state: () => ({ count: 0 }),
  getters: {
    double: (state) => state.count * 2,
  },
  actions: {
    increment() {
      this.count++
    },
  },
})

                                                可以认为 state 是 store 的数据 (data)，
                                                getters 是 store 的计算属性 (computed)，
                                                而 actions 则是方法 (methods)。

                                                # Setup Store

                                                相对于选项式store，也可以使用组合式的store

                                                export const useCounterStore = defineStore('counter', () => {
  const count = ref(0)
  function increment() {
    count.value++
  }

  return { count, increment }
})

                                                在 Setup Store 中：

                                                • ref() 就是 state 属性
                                                • computed() 就是 getters
                                                • function() 就是 actions
                                                  喜欢哪种store，就使用哪种store即可。

                                                # 使用store

                                                store 是一个用 reactive 包装的对象，这意味着不需要在 getters 后面写 .value

                                                • 当从store中进行解构的时候，会破坏其中的响应式
                                                <script setup>
  const store = useCounterStore()
  // ❌ 这将不起作用，因为它破坏了响应性
  // 这就和直接解构 `props` 一样
  const { name, doubleCount } = store 
  name // 将始终是 "Eduardo" 
  doubleCount // 将始终是 0 
  setTimeout(() => {
    store.increment()
  }, 1000)
  // ✅ 这样写是响应式的
  // 💡 当然你也可以直接使用 `store.doubleCount`
  const doubleValue = computed(() => store.doubleCount)
</script>
                                                • 如果通过解构的形式提取属性，需要使用storeToRefs(),来保持响应
                                                <script setup>
import { storeToRefs } from 'pinia'
const store = useCounterStore()
// `name` 和 `doubleCount` 是响应式的 ref
// 同时通过插件添加的属性也会被提取为 ref
// 并且会跳过所有的 action 或非响应式 (不是 ref 或 reactive) 的属性
const { name, doubleCount } = storeToRefs(store)
// 作为 action 的 increment 可以直接解构
const { increment } = store
</script>

                                                # STATE

                                                # 定义STATE

                                                在store中定义state，state用于存储数据

                                                import { defineStore } from 'pinia'

const useStore = defineStore('storeId', {
  // 为了完整类型推理，推荐使用箭头函数
  state: () => {
    return {
      // 所有这些属性都将自动推断出它们的类型
      count: 0,
      name: 'Eduardo',
      isAdmin: true,
      items: [],
      hasChanged: true,
    }
  },
})

                                                # 访问STATE

                                                默认情况下，你可以通过 store 实例访问 state，直接对其进行读写

                                                const store = useStore()

store.count++

                                                # 重置STATE

                                                你可以通过调用 store 的 $reset() 方法将 state 重置为初始值

                                                const store = useStore()

store.$reset()

                                                # 变更STATE

                                                除了用 store.count++ 直接改变 store，你还可以调用 $patch 方法。
                                                它允许你用一个 state 的补丁对象在同一时间更改多个属性

                                                store.$patch({
  count: store.count + 1,
  age: 120,
  name: 'DIO',
})

                                                也可以用过方法的形式进行修改state

                                                store.$patch((state) => {
  state.items.push({ name: 'shoes', quantity: 1 })
  state.hasChanged = true
})

                                                # Getter

                                                # 基础使用

                                                Getter 完全等同于 store 的 state 的计算值。
                                                可以通过 defineStore() 中的 getters 属性来定义它们。推荐使用箭头函数
                                                它将接收 state 作为第一个参数

                                                export const useStore = defineStore('main', {
  state: () => ({
    count: 0,
  }),
  getters: {
    doubleCount: (state) => state.count * 2,
  },
})

                                                # 访问其他getter

                                                大多数时候，getter 仅依赖 state，不过，有时它们也可能会使用其他 getter。
                                                因此，即使在使用常规函数定义 getter 时，我们也可以通过 this 访问到整个 store 实例
                                                这里通过 this.doubleCount 来访问另一个getter

                                                export const useStore = defineStore('main', {
  state: () => ({
    count: 0,
  }),
  getters: {
    // 自动推断出返回类型是一个 number
    doubleCount(state) {
      return state.count * 2
    },
    // 返回类型**必须**明确设置
    doublePlusOne(){
      // 整个 store 的 自动补全和类型标注 ✨
      return this.doubleCount + 1
    },
  },
})

                                                在这里就能访问到getter中的值了

                                                <script setup>
import { useCounterStore } from './counterStore'
const store = useCounterStore()
</script>
<template>
  <p>Double count is {{ store.doubleCount }}</p>
</template>

                                                # 向 getter 传递参数

                                                Getter 只是幕后的计算属性，所以不可以向它们传递任何参数。不过，你可以从 getter 中返回一个函数，该函数可以接受任意参数

                                                export const useStore = defineStore('main', {
  getters: {
    getUserById: (state) => {
      // 通过在这里返回一个函数，才函数中传参，来达到添加参数的目的
      return (userId) => state.users.find((user) => user.id === userId)
    },
  },
})

                                                具体的使用方法,等于就是在方法里面传参了

                                                <script setup>
import { useUserListStore } from './store'
const userList = useUserListStore()
const { getUserById } = storeToRefs(userList)
// 请注意，你需要使用 `getUserById.value` 来访问
// <script setup> 中的函数
</script>

<template>
  <p>User 2: {{ getUserById(2) }}</p>
</template>

                                                # 使用 setup() 时的用法

                                                <script setup>
const store = useCounterStore()
store.count = 3
store.doubleCount // 6
</script>

                                                # Action

                                                Action 相当于组件中的 method。它们可以通过 defineStore() 中的 actions 属性来定义
                                                在actions中直接写方法即可，其实只是把methods换成了actions

                                                export const useStore = defineStore('main', {
  state: () => ({
    count: 0,
  }),
  actions: {
    increment() {
      this.count++
    },
    randomizeCounter() {
      this.count = Math.round(100 * Math.random())
    },
  },
})

                                                类似 getter，action 也可通过 this 访问整个 store 实例
                                                同时，也可以在actions里面写异步方法，不用像vuex那样分开写了。

                                                import { mande } from 'mande'

const api = mande('/api/users')

export const useUsers = defineStore('users', {
  state: () => ({
    userData: null,
    // ...
  }),

  actions: {
    async registerUser(login, password) {
      try {
        this.userData = await api.post({ login, password })
        showTooltip(`Welcome back ${this.userData.name}!`)
      } catch (error) {
        showTooltip(error)
        // 让表单组件显示错误
        return error
      }
    },
  },
})

                                                # action的调用

                                                Action 可以像函数或者通常意义上的方法一样被调用
                                                只要获取到 store 对象后，直接当方法使用即可。

                                                <script setup>
const store = useCounterStore()
// 将 action 作为 store 的方法进行调用
store.randomizeCounter()
</script>
<template>
  <!-- 即使在模板中也可以 -->
  <button @click="store.randomizeCounter()">Randomize</button>
</template>

                                                # 数据持久化

                                                通过上面的步骤已经完成了数据的存取，但是pinia的数据会在页面刷新后丢失，
                                                这里我们可以通过 pinia 的一个插件进行数据持久化处理。
                                                插件 pinia-plugin-persist 可以辅助实现数据持久化功能。

                                                # 安装引入

                                                npm i pinia-plugin-persist

                                                在main.js中引入

                                                import { createPinia } from 'pinia'//引入pinia
import piniaPluginPersist from 'pinia-plugin-persist'//引入pinia数据持久化插件

app.use(createPinia().use(piniaPluginPersist))//安装插件

                                                # 文件配置

                                                在 stores/index.js 这个文件中进行配置。

                                                • 主要是添加 persist 这一项
                                                • 设置一个key进行存储
                                                • 设置一个存储方式 sessionStorage / localStorage

                                                然后你什么都不用做，pinia的数据刷新就不会丢失了。

                                                import { defineStore } from 'pinia'
export const useStore = defineStore('managestate', {
  persist: {
    enabled: true,//开启数据持久化
    strategies: [
      {
        key: 'managestate',//给一个要保存的名称
        storage: sessionStorage,//sessionStorage / localStorage 存储方式
      }
    ]
  },
  
  state() {}
  },
  getters: {}
  },
  actions: {}
  }
})
